Every HTML page has the same basic structure. A head section for information about the page and a body section for everything users actually see.

Up to this point you have been writing bits of HTML directly and that works fine for learning. But real web pages need a proper structure. Without it the browser might still show your content but it is not professional and things can go wrong.

Think of it like writing an essay. You can just scribble sentences but it is always better when you have a title, an introduction, and clear sections. HTML works the same way.

The Skeleton of Every HTML Page

Every proper HTML page has the same basic skeleton. Every single one. Whether it is a simple practice page or a massive website like Wikipedia, they all start with this.

<!DOCTYPE html>
<html lang="en">
  <head>
    <!-- Information about the page -->
  </head>
  <body>
    <!-- The content users see -->
  </body>
</html>

Let us break down what each part does.

  • <!DOCTYPE html> — tells the browser this is an HTML5 page
  • <html> — wraps the entire document, everything lives inside it
  • <head> — holds information about the page that users do not see directly
  • <body> — the actual content users see, headings, paragraphs, images, everything

DOCTYPE

This line goes at the very top of every HTML file.

<!DOCTYPE html>

It tells the browser "this file is written in HTML5." Without it the browser has to guess what version of HTML you are using and sometimes it guesses wrong. Always include this at the very top.

The html Tag

This wraps the entire page. Everything else lives inside it.

<html lang="en">

The lang="en" attribute tells the browser, screen readers, and search engines that the page is in English. If your page is in Spanish you would write lang="es". It is a small detail but it matters for accessibility and SEO.

The head Section

Think of the head as the settings of your page. Users do not see it directly but it is crucial. Here is what goes inside it.

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="A page about Nadia, software engineer and educator.">
  <meta name="author" content="Your Name">
  <title>Nadia</title>
</head>

Let us go through each one.

  • <meta charset="UTF-8"> — tells the browser what character set to use. UTF-8 supports pretty much every language including emojis. Without it weird characters can show up instead of letters.
  • <meta name="viewport" ...> — makes your page mobile friendly. Without it your site could look squished or zoomed out on phones.
  • <meta name="description" ...> — a short description of your page. Search engines use this in search results under your link.
  • <meta name="author" ...> — optional. Just credits you as the author of the page.
  • <title> — shows up in the browser tab and is what search engines display as the clickable headline in results.
💡

The head is not visible on the page but it is what makes your page work properly on mobile, show up correctly in search results, and display the right tab title.

The body Section

Everything the user actually sees goes inside the body. All the things you have been learning up to this point go here.

<body>
  <h1>Nadia</h1>
  <p>Nadia is a software engineer and coding teacher.</p>
  <img src="nadia.png" width="250" height="auto" alt="Photo of Nadia">
</body>

Headings, paragraphs, images, links, lists, tables, forms — all of it goes inside the body.
