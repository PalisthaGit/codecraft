HTML forms let users enter data and send it to a server. They're how login pages, search bars, contact forms, and checkouts work. The <form> element is the container; <input>, <textarea>, and <button> go inside it.

Almost every web app needs forms. Whether it's a search box, a sign-up page, or a settings panel — forms are how users communicate with a site.

The Form Element

The <form> element wraps all your form controls. The action attribute sets where the data goes; method sets how it's sent.

<form action="/submit" method="post">
  <!-- inputs go here -->
</form>

Text Inputs

The most common form element. The type attribute controls what kind of input it is:

<input type="text" name="username" placeholder="Enter your name">
<input type="email" name="email" placeholder="you@example.com">
<input type="password" name="password" placeholder="Password">
<input type="number" name="age" min="1" max="120">

Labels

Always pair inputs with <label> elements. The for attribute should match the input's id. This makes the label clickable and improves accessibility:

<label for="email">Email address</label>
<input type="email" id="email" name="email">
💡

Skipping labels is one of the most common accessibility mistakes. Screen readers use labels to describe inputs to users who can't see the screen. Always include them.

Textarea

For multi-line text input, use <textarea>:

<label for="message">Message</label>
<textarea id="message" name="message" rows="4" placeholder="Write your message..."></textarea>

Select Dropdowns

<label for="country">Country</label>
<select id="country" name="country">
  <option value="">Select a country</option>
  <option value="us">United States</option>
  <option value="uk">United Kingdom</option>
  <option value="ca">Canada</option>
</select>

Checkboxes and Radio Buttons

<!-- Checkboxes — multiple can be selected -->
<label>
  <input type="checkbox" name="terms" value="agreed">
  I agree to the terms
</label>

<!-- Radio buttons — only one can be selected -->
<label>
  <input type="radio" name="plan" value="free"> Free
</label>
<label>
  <input type="radio" name="plan" value="pro"> Pro
</label>

Submit Button

<button type="submit">Send Message</button>

<!-- Or using input -->
<input type="submit" value="Send Message">

Required Fields and Validation

HTML has built-in validation attributes that work without any JavaScript:

<input type="email" name="email" required>
<input type="text" name="username" required minlength="3" maxlength="20">
<input type="number" name="age" min="18" max="99">

A Complete Form Example

<form action="/contact" method="post">
  <label for="name">Name</label>
  <input type="text" id="name" name="name" required>

  <label for="email">Email</label>
  <input type="email" id="email" name="email" required>

  <label for="message">Message</label>
  <textarea id="message" name="message" rows="4" required></textarea>

  <button type="submit">Send</button>
</form>
