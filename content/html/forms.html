A form is a way for users to give information to a page. HTML gives us the <form> tag to wrap all our inputs together.

You have seen plenty of websites up to now.

Think about the last time you searched for something online. You clicked on a box, typed something in, and hit enter. The site took what you typed and did something with it.

That is a form. A form is simply a way for you to give information and for the site to use that information to do something.

See that search box at the bottom of the card? That is a form. You type something in, press search, and the page uses what you typed to find results.

Let us learn how to build one.

The form Tag

All HTML forms are wrapped inside a <form> tag. That is the container. Inside it you decide what kind of inputs you want.

To create a text box you use the <input> tag with type="text".

But if you look at a real search box there is usually some grey hint text inside it. That is called a placeholder. You add it with the placeholder attribute.

Now the box shows that hint text until the user starts typing.

Adding a Submit Button

Now let us add a button so the user can actually submit the form. For that you use another <input> but this time with type="submit". You can give it a label using the value attribute.

Now you have a text box where the user can type, a placeholder that guides them, and a button that lets them send the form.

💡

For now this form will not actually search anything or send data anywhere. That requires a backend. The goal here is just to learn how to build the form itself.

Radio Buttons

Sometimes instead of letting users type freely you want to give them a fixed set of options to choose from. That is where radio buttons come in. The user can only pick one option at a time.

Each radio button needs three things:

  • type="radio" — tells the browser this is a radio button
  • name="group-name" — groups them together so only one can be selected
  • value="something" — the value that gets saved when selected
💡

Same name means same group. Only one option in the group can be picked at a time.

Dropdown

Sometimes you want a neat menu where the user clicks and chooses from a list. That is a dropdown. Instead of showing all options at once like radio buttons you put them in one tidy menu.

You use two tags together. <select> creates the dropdown box and <option> is each choice inside it.

Textarea

A regular text box is perfect for short input like a name or a search term. But what if you want the user to write more? A comment, some feedback, or a longer answer.

That is where <textarea> comes in. It creates a bigger box where users can type multiple lines. You control the size with rows and cols attributes.

💡

<input type="text"> is for one line only. <textarea> lets users type multiple lines and even expand the box.

The action Attribute

In real forms you will usually see a action attribute on the <form> tag. This tells the form where to send the data when the user clicks submit.

<form action="/submit">
  <input type="text">
  <input type="submit" value="Send">
</form>

When the user clicks Send the browser sends all the form data to that location. That could be a page, a file, or a server.

If you are just learning and do not have a backend yet that is completely fine. You can leave the action empty or leave it out entirely and the form will still work for building and testing.

💡

Leave action="" or just use <form> without an action while you are learning. You can always add the real destination later.
